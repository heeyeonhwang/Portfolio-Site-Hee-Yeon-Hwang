Airport Software Re-Design

Design Challenge for a Specialty Airport Services Software

1. The Challenge

For a week long design challenge, I re-designed a web-based airport software ( X 1 FBO) that managess services to airplanes. My goal was to eliminate unnecessary clicks for users from when they created a new request to closing a request and create a consistent visual design that matched the other modules in the software.

The main challenge of this project was that this was a specialty software for the aviation industry and I was not familiar with the users' experiences. I also only had one contact to conduct both user research and usability testing.

2. Research

My first step was to understand the users' experience since this was my first time learning about the software's industry. I conducted my user interview with my only contact, who was a customer service manager in the industry. I asked her about the step-by-step workflow of creating a new service request to closing the request. Then in order to find painpoints, I asked the likes and frustration about the each step of the process. For each problem, I prioritized, created possible solutions, and created a list.

The main changes were to create a new user flow to eliminate uncessary steps for easier and quicker design, such as re-formatting the options on each page to stop additional clicks to get to new features. For example, users could close a service request right away after the airplane departs and they mark the time of departure. Before, they would have to save and then click three more times to close the request.

3. Wire-framing & Prototyping

The initial wire-frames were created with paper and pen before moving to Adobe XD for a high-fidelity prototype.

4. Usability Testing

I conducted a usability testing with the same contact in the aviation industry. With the XD prototype, I first asked open-ended question about the impressions and thoughts of the new layout. Then, I created four different tasks that reflected each step of the user flow to complete with the prototype. The main feedback was that it was difficult to figure out how to create a new invoice and see the "Save & Close Request" button to close the request. So I changed the buttons to a brighter color.

5. Visual Design & Comparisons

For the visual design, I wanted to create a consistent layout that matched the other modules of the software to help users' usability tranfser easily from module to module. This airport software has about five different modules with varying visual design choices with inconsitent styles. Since there was one module called SinglePoint, that had the most polished design out of the five, I utilized the dark colors and components to set as a guideline for the software.

In addition to the overall change in the design theme, I added UI changes from the painpoints that I gathered from the user interview. I condensed the columns and expanded the row space for users to see all of the information without scrolling all the time. I also condesned the services and the concierge section in the new request page for a bigger summary section. I create visual design changes that are backed by the user research data.
